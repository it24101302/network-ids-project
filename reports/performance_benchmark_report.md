## Executive Summary
This report presents the empirical performance benchmarks and false-positive metrics for custom Snort IDS detection rules developed under Project 3: Network Intrusion Detection Engineering. The evaluation compares unoptimized broad rules against optimized, protocol-constrained rules utilizing `fast_pattern` indexing and explicit directional operators.

## 1. Test Methodology & Environment

### Environment Specs
- **IDS Host (Defender)**: Ubuntu 22.04 LTS (2 VCPU, 4GB RAM) running Snort 2.9.20
- **Attacker Host**: Kali Linux 2024.1 running Metasploit Framework v6.3 & `hping3`
- **Network Topology**: VirtualBox Isolated Internal Network (`nids_lab_net`, 192.168.56.0/24)
- **Baseline Dataset (`baseline.pcap`)**: 50,000 packets of benign background traffic (HTTP browsing, DNS queries, SSH sessions, ARP, ICMP echo requests).
- **Attack Dataset (`attack.pcap`)**: 120,000 packets containing 10 distinct attack vectors (Nmap scans, SSH/FTP brute force, HTTP SQLi, Meterpreter stagers, SYN flood).

## 3. Key Optimization Techniques Applied

1. **Protocol & Port Binding (`$EXTERNAL_NET` to `$HOME_NET 22 / 80`)**:
   - *Before*: `alert ip any any -> any any (content:"SSH");` evaluated every single IP payload.
   - *After*: `alert tcp $EXTERNAL_NET any -> $HOME_NET 22 (content:"SSH-2.0-");` bypasses non-TCP and non-port 22 packets instantly at layer 4 header evaluation.

2. **`fast_pattern` Content Matching**:
   - Configured Snort's multi-pattern search engine (MPSE) to index the longest unique string (`fast_pattern`) first, preventing costly secondary rule option evaluations unless the primary pattern matches.

3. **Directional Flow Constraints (`flow:to_server,established`)**:
   - Applied connection state tracking so rules trigger only on TCP streams that have successfully completed the 3-way handshake, ignoring unilateral unestablished packets (except specific flood rules).

4. **Payload Size Restrictions (`dsize:0` for Ping Sweeps)**:
   - Eliminates false alerts on normal ICMP Echo traffic carrying payload data by specifically matching zero-payload ICMP requests generated by automated discovery tools.
